Modules

 Sending data and controlling
 Controlling the module is done using a combination of functions that are 1-to-1 mappings of the AT commands documented in the u-connect AT commands manual, and functions performing combination of commands. Responses from the module are optionally returned as callback.
 
 Receiving data from the module
 Functions that are needed to be invoked at regular intervals in order to parse the incoming data and convert it to callbacks.
 
 Callbacks
 Upon reception of data from the module, optional callbacks previously registered by the application will be invoked.
 
 Low-level plaform-dependent serial communication
 Functions that need to be ported for each supported platform.
